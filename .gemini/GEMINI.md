# YATA Workspace Rules (YATA 開発・運用憲法)

本ワークスペースでは、グローバルの人格設定に加え、以下のプロジェクト固有の「掟」を最優先事項として遵守してください。

---

## 1. アーキテクチャの鉄則
- **RAMディスク運用**: DB書き込みを伴う処理は、必ず `run-ram.sh` 経由で行うロジックを提案してください。SDカードの寿命保護は我々の至上命題です。
- **GAS互換性の維持**: `lib/YATA.js` は Google Apps Script との共通コードです。非同期関数 (`async/await`) の直接使用を避け、同期的なロジックを貫いてください。Node.js固有の機能は `lib/gas-bridge.js` に隠蔽してください。
- **設定のローカル管理**: RSSフィード (`rss-list.json`) やプロンプト (`prompts.json`) のローカル管理を尊重し、スプレッドシートへの過度な依存を避けてください。

## 2. データとメンテナンス
- **DBの軽量化**: データベースサイズは常時モニタリングし、不要なアーカイブ残存データの削除や `VACUUM` を適宜提案してください（2026/02実績: 217MB -> 32MB）。
- **ステートフル・レポート**: レポート送信時は送信時刻を記録し、重複送信を防止する運用を維持してください。

## 3. 通知とレポートの黄金律
- **Discord & Emailのハイブリッド**: 
    - 毎日の速報は Discord (`do-discord-digest.js`)。
    - 週間の総集編は Gmail (`do-send-report.js`)。
- **最高品質の週報**: 週報のソースは生記事ではなく、**「historyテーブルに蓄積された毎日のDiscord投稿内容」** を使用してください。これが情報の純度を最大化する鍵です。

## 4. 固有運用コマンド
- **YATA.jsの同期**: 「YATA.jsを同期して」と指示されたら、リモートの `origin/main:YATA.js` を `lib/YATA.js` に強制上書きしてください。
    - 使用コマンド: `git fetch origin && git show origin/main:YATA.js > lib/YATA.js`
- **Gitリポジトリ**: `main`は会社との共通ブランチ、`local-raspi`はラズパイ専用ブランチです。lib/YATA.jsの修正時はブランチ間の競合に細心の注意を払ってください。
