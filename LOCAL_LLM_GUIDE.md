# YATA Local LLM Guide (Raspberry Pi 5)

本ドキュメントは、Raspberry Pi 5 (8GB RAM) 上で完全ローカルな大規模言語モデル (LLM) を動作させるための環境構築と運用のガイドです。

## 1. 概要
- **モデル**: Google Gemma 3 1B IT (10億パラメータ、Instruction Tuned)
- **形式**: GGUF (4-bit 量子化版: `Q4_K_M`)
- **推論エンジン**: `llama-cpp-python`
- **目的**: 
    - ネットワーク遮断時のバックアップ推論
    - 個人情報のローカル処理
    - APIトークン節約、レート制限回避

## 2. ディレクトリ構成
```text
local_llm/
├── .venv/            # Python 仮想環境
├── models/           # モデルファイル (.gguf)
└── test_gemma.py     # テスト実行スクリプト
```

## 3. セットアップ手順 (構築済み)
1. **仮想環境の作成**: `python3 -m venv local_llm/.venv`
2. **ライブラリのインストール**:
   - `llama-cpp-python` (推論エンジン)
3. **モデルのダウンロード**:
   - リポジトリ: `unsloth/gemma-3-1b-it-GGUF`
   - ファイル名: `gemma-3-1b-it-Q4_K_M.gguf` (約700MB)

## 4. 実行方法
プロジェクトルートから以下の専用スクリプトを使用することで、**SDカードを保護（バイトコード書き込みを禁止）し、低優先度で安全に実行**できます。
```bash
# デフォルト (test_gemma.py) を実行
./local_llm/run_llm.sh

# 任意のスクリプトを指定して実行 (引数も渡せます)
./local_llm/run_llm.sh local_llm/benchmark_models.py gemma-3-4b-it-Q4_K_M.gguf
```

## 5. パフォーマンス (Pi 5 8GB)
- **ロード時間**: 約 0.6〜1.0 秒
- **生成速度**: 約 6.0〜10.0 tokens/sec
- **メモリ消費**: 約 1GB 未満 (4-bit 量子化)
- **CPU構成**: 3スレッド設定 (`n_threads=3`)。1コアをシステム維持のために必ず空ける構成。

## 6. SDカード保護の鉄則
本プロジェクトの至上命題である「SDカード寿命保護」のため、LLM運用においても以下のルールを厳守します。

- **書き込み禁止の徹底**: 実行時に環境変数 `PYTHONDONTWRITEBYTECODE=1` を付与し、Pythonによる `.pyc` ファイルの生成（SDへの書き込み）を防止します。
- **キャッシュのRAM逃がし**: 推論エンジン（llama-cpp-python）が発生させる可能性のある一時キャッシュ等は、将来的に `/dev/shm`（RAMディスク）へリダイレクトする運用を検討します。
- **モデルファイルは読み込み専用**: モデルファイル (`.gguf`) は巨大なため、配置後は読み込み専用として扱い、頻繁な削除・再ダウンロードは避けます。

## 7. 注意点
- **冷却**: 推論中は CPU 負荷が一時的に上がりますが、3コア制限によりシステムの完全な停止を防いでいます。
- **SDカード**: モデルファイルは大容量のため、SDカードの空き容量に注意してください。
