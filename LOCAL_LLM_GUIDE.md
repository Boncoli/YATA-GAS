# YATA Local LLM Guide (Raspberry Pi 5)

本ドキュメントは、Raspberry Pi 5 (8GB RAM) 上で完全ローカルな大規模言語モデル (LLM) を動作させるための環境構築と運用のガイドです。

## 1. 概要
- **メインモデル**: Google Gemma 3 4B IT (4-bit 量子化版: `Q4_K_M`)
- **サブモデル**: Google Gemma 3 1B IT (軽量版、緊急用)
- **推論エンジン**: `llama-cpp-python` (OpenAI 互換 HTTP サーバー)
- **実行形式**: PM2 による常駐サーバー (yata-ai-server)
- **ポート**: `8000` (localhost)
- **目的**: 
    - **爆速レスポンス**: ロード時間をゼロにし、数秒での返答を実現。
    - **プライバシー保護**: 位置情報や会話履歴を一切外部に出さずに処理。
    - **状況把握**: ニュース、天気、自身のCPU温度を考慮した高度な対話。

## 2. ディレクトリ構成
```text
local_llm/
├── .venv/            # Python 仮想環境 (llama-cpp-python[server] 導入済み)
├── models/           # モデルファイル (.gguf)
├── run_server.sh     # 【New】常駐サーバー起動スクリプト
├── chat_gemma.py     # 【New】対話用推論スクリプト (単発実行用)
└── test_gemma.py     # テスト実行スクリプト
```

## 3. セットアップと運用 (2026/03/09 刷新)
1. **サーバー起動**:
   ```bash
   # PM2 で管理 (自動起動設定済み)
   pm2 start local_llm/run_server.sh --name yata-ai-server
   ```
2. **Discord 連携**:
   `modules/discord-bot.js` が `http://localhost:8000/v1/chat/completions` を叩き、数秒で返答を取得します。
3. **安全策 (Failsafe)**:
   - **温度制限**: CPU 温度が 75℃ を超えると、ローカル推論をスキップしクラウド (OpenAI/Gemini) へフォールバックします。
   - **タイムアウト**: 40 秒以上応答がない場合もクラウドへ逃げます。

## 4. パフォーマンスと安全性 (Pi 5 8GB)
- **ロード時間**: 0 秒 (常駐時)
- **生成速度**: 約 6.0〜8.0 tokens/sec (Gemma 3 4B)
- **メモリ消費**: 約 3.2GB (モデルロード時)
- **待機時CPU**: ほぼ 0%
- **CPU構成 (鉄壁の保護)**: 
    - **物理コア隔離**: `taskset -c 0-2` により、4コアのうち3コアのみをAIに使用。**コア3はシステム・Cron・DB同期専用の「聖域」として完全開放。**
    - **スレッド制限**: `n_threads=3` 指定。
    - **優先度**: `nice -n 15` でバックグラウンド実行を徹底し、OSの動作を最優先。

## 5. 高速化とSDカード保護の鉄則
- **プロンプト・キャッシング**: 固定内容（ペルソナ、掟）を冒頭に、変動内容（状況コンテキスト）を後半に配置し、Llama.cppのKVキャッシュ効率を最大化。
- **常駐化による摩耗低減**: 毎回モデルを読み出す I/O 負荷をなくし、メモリ上でのみ処理を完結させます。
- **書き込み禁止**: サーバー実行時も `PYTHONDONTWRITEBYTECODE=1` を維持。


## 6. SDカード保護の鉄則
本プロジェクトの至上命題である「SDカード寿命保護」のため、LLM運用においても以下のルールを厳守します。

- **書き込み禁止の徹底**: 実行時に環境変数 `PYTHONDONTWRITEBYTECODE=1` を付与し、Pythonによる `.pyc` ファイルの生成（SDへの書き込み）を防止します。
- **キャッシュのRAM逃がし**: 推論エンジン（llama-cpp-python）が発生させる可能性のある一時キャッシュ等は、将来的に `/dev/shm`（RAMディスク）へリダイレクトする運用を検討します。
- **モデルファイルは読み込み専用**: モデルファイル (`.gguf`) は巨大なため、配置後は読み込み専用として扱い、頻繁な削除・再ダウンロードは避けます。

## 7. 注意点
- **冷却**: 推論中は CPU 負荷が一時的に上がりますが、3コア制限によりシステムの完全な停止を防いでいます。
- **SDカード**: モデルファイルは大容量のため、SDカードの空き容量に注意してください。
