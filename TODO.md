# YATA Project TODO & Ambitions

YATA (boncoli RasPi) をさらに進化させ、真の「パーソナル・AI・パートナー」へと昇華させるためのロードマップ。

## 📊 次なるライフログ・パーソナライズ (Next Evolution)
- [ ] **デジタル・ツイン (分身) の構築と深化**:
    - 「独り言キャッチャー」で得た生の思考プロセスを Gemini で分析し、BON 様の価値観や判断基準をプロファイリング。
    - ニュースや事象に対し「BON 様ならどう考え、どう反応するか」をシミュレート・代弁できるレベルまで精度を上げる。
- [ ] **興味関心ベースの「意味的マッチング」向上**:
    - 現在のキーワード＋AI要約ピックアップをさらに強化。ベクトル検索を使い、キーワードが直接一致しなくても「文脈的に近い」ニュースを確実に捉える。
- [ ] **Apple Watch 連携の安定化**:
    - 外出先からの Watch 単体送信の成功率向上（リトライ処理の実装）。
- [ ] **ローカル LLM (Gemma 3) の実用化**:
    - [x] **ローカル独り言の自律運用**: `tasks/do-local-mutter.py` を実装し、1時間おき（時報）の自律的な「ボヤキ」を開始。Gemma 3 4Bによる高度なデジタルツイン発話を実現。
    - [ ] **プライバシー重視の分析**: `voice_catcher` で得た独り言を、外部 API を通さずローカルで感情分析するモードの実装。
    - **オフライン・バックアップ**: ネットワーク遮断時に自動でローカル LLM へ切り替えるフェイルセーフの設計。
- [ ] **ローカル音声合成 (TTS)**:
    - 朝のDiscord要約を、ラズパイが自室や車内で読み上げ。

## 🗺️ 地図の進化 (Map Evolution)
- [ ] **タイムライン・フィルタリングの高度化**:
    - 指定した期間の軌跡のみを表示し、その期間の総走行距離や平均燃費も併記する。
- [ ] **ヒートマップ表示**:
    - よく通る道（CX-80の活動圏内）を視覚的に強調。

- [x] **YATA.js 2nd Vectorization (Method Embedding) の実装と 256次元圧縮による極限最適化 (2026/03/04)**:
    - 記事の「話題 (Topic)」に加え、「測定手法・原理 (Method)」に特化した第2ベクトルを生成・保存する機能を実装。
    - 異分野間で同じ手法が使われ始めた「技術パラダイムの兆候」を検知できる高度な分析基盤（R&D特化）を確立。
    - `text-embedding-3-small` のネイティブ機能（Matryoshka Representation Learning）を利用し、ベクトル次元を1536→256次元へ圧縮。精度劣化を数%に抑えつつGASのデータ容量を1/6に激減させた。
- [x] **ヘルスケアデータ同期の信頼性向上 (2026/03/03)**:
    - iPhoneショートカットからのデータ受信時に発生していたSQLパラメータ不足エラーを、名前付きパラメータ（`:date`, `:type`等）の導入により解消。
- [x] **デジタルツインの高度な統合とローカルLLM（Gemma 3 4B）の自律運用開始 (2026/03/03)**:
    - 複数モデル（GPT-5, 27B等）の解析結果を `synthesized_master_persona.md` に集約。ボヤキ生成のコアエンジンとして搭載。
    - 3コア制限（n_threads=3）をかけた `tasks/do-local-mutter.py` を実装。システム負荷を抑えつつ、直近24時間のニュースと動静に基づいた「自然なボヤキ」を1時間おきにDiscordへ放流開始。
- [x] **DB軽量化と長期運用体制の確立 (2026/03/03)**:
    - `tasks/archive-old-articles.js` を動的日付計算（180日）に修正。半年サイクルでの自動アーカイブ・VACUUMを可能にした。
- [x] **ダッシュボードの視覚的品質改善 (2026/03/03)**:
    - フォールバック文字変換（normalize_text）を `server.js` に実装。全角記号による「豆腐化（□）」を根絶。
- [x] **リポジトリの整理 (2026/03/03)**:
    - 大容量JSON生データをGit追跡から除外、動的な `interests.json` も追跡除外。運用ドキュメントに「プロセス・マップ」を追記し、全体像の視認性を向上。
- [x] **YATA.js APIコスト浪費の無限ループ修正とプロンプト厳格化 (2026/03/02)**:
    - 2月の月またぎ（28日問題）に起因する、ベクトル削除（1ヶ月前）と生成（30日前）の閾値競合による「無限再生成ループ」を解消。削除を35日前に固定。
    - APIエラー時の無限リトライ防止ロジックと、最新の厳格・客観的事実抽出型プロンプトを導入。
    - `gpt-5-nano` の推論量を `minimal` から `low` に微調整。
- [x] **AI独り言の情緒強化と個人情報マスク処理 (2026/03/02)**:
    - 発話リズムを時計の5分刻みに同期させ、より自然な存在感を演出。
    - 外部APIへのプロファイル送信時に特定個人情報（地域・会社）を自動マスクするセキュリティ機能を実装。
- [x] **ローカルLLM (Gemma 3 4B) による長時間バックグラウンド分析 (2026/03/01)**:
    - 18万件のアーカイブデータを100件ずつのチャンクに分割し、状態を記録しながら一晩かけて処理する「ダラダラ分析スクリプト」を実装。
    - 熱暴走対策のクールダウン機能とレジューム機能を搭載し、ローカルでの持続的な深層プロファイリング環境を確立。
