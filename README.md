# YATA (八咫) - AI Intelligence Grimoire
> **The Three-Legged Guide to the Web.**
> **情報の海を導き、真実を映し出す。あなたのための「AIインテリジェンス・パートナー」。**

本書は、AI駆動型RSS収集・分析プラットフォーム「YATA」の全貌を記したマスターマニュアル（虎の巻）である。開発の背景、内部構造、そしてメンテナンス時の急所を網羅している。

---

## ⛩️ コンセプト：三本足の導き手
名前の由来は、日本神話の「八咫烏（ヤタガラス）」と「八咫鏡（ヤタノカガミ）」。
1.  **収集 (Collection)**: 広大なWebから鮮度の高い情報を掴む足。
2.  **分析 (Analysis)**: 本質を見抜き、過去からの文脈を紡ぐ足。
3.  **伝達 (Dispatch)**: 必要な時に、必要な形（メール/Web）で届ける足。

この3つの機能を調和させ、情報のノイズを 90% 削減しつつ、意思決定に必要なインサイトを抽出することが本システムの使命である。

---

## 🚀 このツールで「できること」

### 1. インテリジェントな自動巡回
*   **多層監視**: ユーザーが「RSS」シートに登録したソース（技術、論文、ビジネス、臨床等）を24時間体制で監視。
*   **スマートな重複排除**: URLの正規化とタイトルの一致確認により、同じニュースを二度読ませない。

### 2. 【NEW】予兆（サイン）検知：核形成（Nucleation）の特定
*   **マジョリティからの乖離**: 現在の主流なトレンド（情報の重心）を数学的に特定し、そこから遠く離れた場所にある「孤独な点（アウトライヤー）」を抽出。
*   **ドメイン横断の同期**: 異なる専門分野（ドメイン）のソースから、同じ「異質な概念」が同時に語られ始めた瞬間を「核形成」として検知。
*   **自律的インサイト**: 誰かがキーワードを検索する前に、AI自らが「何かが起きようとしている」ことを察知し、将来のシナリオを推論して報告する。

### 3. AIによる「意味」の理解（ハイブリッド検索）
*   **セマンティック検索**: ベクトル変換（Embedding）により、言葉の表面的な一致を超えた「概念の近似」で記事をヒットさせる。
*   **高度なクエリ演算**: AND/OR/NOTやカッコを用いた複雑なキーワード検索にも対応。

### 4. 歴史を汲み取るトレンド分析
*   **記憶を持つレポート**: 過去の分析結果を `DigestHistory` シートに記録。AIが「先週からの進捗」や「継続的な課題」を認識し、単なるニュースの羅列ではない「ストーリー」としてレポート化する。

## 🛠️ メンテナンス・ガイド（虎の巻）

### 1. 設定の司令塔：`AppConfig`
*   **予兆検知の感度**: `SignalDetection` セクションの `OUTLIER_THRESHOLD`（主流からの距離）や `NUCLEATION_RADIUS`（核の密度）を調整することで、インテリジェンスの「鋭さ」を制御できる。
*   **検索スコア**: `System.SearchScore` で、新しい記事をどれだけ優先するか、抜粋の長さをどれだけ評価するかを定義。

### 2. 知能の優先順位：マルチティア・フォールバック
`LlmService` は **Azure OpenAI → OpenAI (Native) → Gemini** の順で自動的に切り替わる。実行環境が `PERSONAL` か `COMPANY` かによって、好みのAPIを優先させることが可能。

### 3. 健康診断：`runAllTests()`
修正後は必ず実行すること。特に新設された `_test_EmergingSignalEngine()` は、ベクトル空間上での重心算出や核形成検知が数学的に正しいかを検証する。
