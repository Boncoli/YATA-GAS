# RSScollect - Google Apps ScriptによるRSSフィード収集・AI見出し生成・週次ダイジェスト

## 概要
このGoogle Apps Script (GAS) プロジェクトは、指定されたRSSフィードから記事を自動的に収集し、Googleスプレッドシートに保存します。さらに、収集した記事の抜粋やタイトルを基に、AI (Gemini, Azure OpenAI等) を利用して「ネットニュース風の見出し」を生成・追記します。

そして、週に一度、収集した記事から重要度や関連性の高いものをAIが自動で選別・要約し、見やすい**HTML形式のダイジェスト**としてメールで配信します。

## 機能
- **RSSフィード収集**: 登録されたRSSフィードから最新記事を自動取得し、重複を排除してスプレッドシートに追記します。
- **AI見出し生成**: 記事の抜粋やタイトルから、AIがキャッチーで簡潔な日本語の見出しを生成します。
  - 短い記事や抜粋がない場合は、タイトルを基に自動翻訳（英語の場合）またはそのまま見出しとして利用します。
  - AIモデルはAzure OpenAI, OpenAI, Geminiから選択可能です。
- **週次ダイジェスト (HTMLメール)**: 過去N日間の記事から重要度を算出し、上位記事をまとめたダイジェストを作成します。
  - 重要度はヒューリスティックなスコアリングと、オプションでAIによる重み付けが可能です。
  - ダイジェストは**HTML形式**で生成され、見やすいレイアウトでメール送信されます。
- **HTMLタグ除去**: 記事の抜粋から不要なHTMLタグを除去し、クリーンなテキストとして扱います。
- **スプレッドシートの自動ソート**: 記事追加後、スプレッドシートを日付の降順で自動的にソートします。

## 週次ダイジェストの構成とデザイン
週次で配信されるHTMLメールは、以下のロジックとデザインで構成されています。

### ロジックのフロー
1. `weeklyDigestJob`が週次トリガーで実行されます。
2. 過去N日間の記事から、専門キーワード（例: 「臨床検査」「AI」「ゲノム」など）に基づいて関連性の高い記事を抽出します。
3. ルールベースのスコア（鮮度、ソース、キーワード等）とAIによるスコア（オプション）を組み合わせて、記事を総合的にランク付けします。
4. LLM（AI）が、ランク付けされた上位記事群を元に、**Markdown形式**でダイジェスト本文（ハイライト、トレンド、Top Picks）を生成します。（このMarkdown生成ロジックは、将来的な拡張のために維持されています）
5. 生成されたMarkdownは、`markdownToHtml`関数によって見やすい**HTML形式**に変換されます。
6. 最終的に`GmailApp`を通じてHTMLメールとして配信されます。

### メールの構成
メールは以下のセクションで構成され、情報を直感的に把握できます。
1. **今週のハイライト**: 全体の動向をまとめた3行の要約。
2. **主要トレンド**: AIが記事を複数のトレンドに分類し、それぞれの要点と代表記事リンクをまとめたセクション。
3. **Top Picks**: 特に注目すべき記事（デフォルト5件）を、AIによる短い要点（TL;DR）付きで紹介。
4. **フッター**: 全記事を確認できるスプレッドシートへのリンク。

### HTMLデザイン
- **フォント**: メイリオやヒラギノ角ゴなど、PCでもスマートフォンでも読みやすい標準的な日本語フォントを使用します。
- **レイアウト**: 白を基調とした背景に、ダークグレーのテキストを配置した、シンプルでクリーンなデザインです。
- **装飾**: リンクは青文字、セクションの区切り線は薄いグレーの線で表現するなど、視覚的に分かりやすいスタイルを適用しています。

## スプレッドシートの構造

### 1. `RSS` シート (RSSフィードリスト)
| 列 | ヘッダー名 | 説明 |
|---|---|---|
| A | サイト名 | RSSフィードの提供元サイト名 |
| B | RSS URL | RSSフィードのURL |

### 2. `collect` シート (収集データ)
| 列 | ヘッダー名 | 説明 |
|---|---|---|
| A | 日付 | 記事の公開日時 (pubDate/updated) |
| B | 元タイトル | 記事の元のタイトル |
| C | URL | 記事のURL |
| D | 抜粋 | 記事の抜粋または概要 (HTMLタグ除去済み) |
| E | 見出し（AI生成）| AIが生成したネットニュース風の見出し |
| F | ソース | 記事のソースサイト名 (RSSシートのサイト名) |
| G | AIスコア | (週次処理) AIが評価した記事の重要度スコア |
| H | AI TL;DR | (週次処理) AIが生成したTop Picks用の1行要約 |


## 設定方法

### 1. Google Apps Script プロジェクトの作成
1. Googleスプレッドシートを開き、「拡張機能」>「Apps Script」を選択してスクリプトエディタを開きます。
2. `RSScollect.js` の内容をスクリプトエディタにコピー＆ペーストします。
3. プロジェクト名を「RSScollect」など、任意の分かりやすい名前に変更します。

### 2. スクリプトプロパティの設定
スクリプトエディタの左側メニューにある「プロジェクトの設定」（歯車アイコン）をクリックし、「スクリプトプロパティ」セクションで以下のプロパティを設定します。

| プロパティ名 | 説明 | 設定例 | 必須/任意 |
|---|---|---|---|
| `GEMINI_API_KEY` | Gemini APIを利用する場合のAPIキー | `AIzaSy...` | AI利用時 |
| `AZURE_ENDPOINT_URL` | Azure OpenAIを利用する場合のエンドポイントURL | `https://your-resource.openai.azure.com/...` | AI利用時 |
| `OPENAI_API_KEY` | Azure OpenAIを利用する場合のAPIキー | `your-azure-openai-api-key` | AI利用時 |
| `OPENAI_API_KEY_PERSONAL` | 個人的なOpenAI APIを利用する場合のAPIキー | `sk-...` | AI利用時 |
| `OPENAI_MODEL_DAILY` | 日次見出し生成で使うOpenAIモデル名 | `gpt-4.1-nano` (既定値) | 任意 |
| `OPENAI_MODEL_WEEKLY` | 週次ダイジェスト生成で使うOpenAIモデル名 | `gpt-4.1-mini` (既定値) | 任意 |
| `DIGEST_DAYS` | 週次ダイジェストの集計日数 | `7` (既定値) | 任意 |
| `DIGEST_TOP_N` | 週次ダイジェストでAI分析の対象とする上位記事数 | `30` (既定値) | 任意 |
| `DIGEST_USE_AI_RANK` | 週次ダイジェストでAIによるバッチ評価（スコア＋要約）を利用するか (`Y`/`N`) | `N` (既定値) | 任意 |
| `DIGEST_AI_CANDIDATES` | AIに渡す最大候補数 (AIランキング/要約利用時) | `50` (既定値) | 任意 |
| `NOTIFY_CHANNEL_WEEKLY` | 週次ダイジェストの通知先 (`email` または `none`) | `email` (既定値) | 任意 |
| `MAIL_TO` | メール通知の送信先メールアドレス (カンマ区切りで複数指定可) | `your-email@example.com` | `NOTIFY_CHANNEL_WEEKLY` が `email` の場合必須 |
| `MAIL_SUBJECT_PREFIX` | メール件名のプレフィックス | `【週間RSS】` (既定値) | 任意 |
| `MAIL_SENDER_NAME` | メール送信者名 | `RSS要約ボット` (既定値) | 任意 |
| `DIGEST_SHEET_URL` | メール末尾に記載するスプレッドシートのURL | `https://docs.google.com/spreadsheets/d/...` | 任意 |
| `TEAMS_WEBHOOK_URL` | **（現在未使用）** MS Teamsへの通知に利用したWebhook URL | `https://outlook.office.com/webhook/...` | 不要 |

**注意:** AIモデルの選択は以下の優先順位で行われます。
1. `AZURE_ENDPOINT_URL` と `OPENAI_API_KEY` が設定されている場合: **Azure OpenAI**
2. `OPENAI_API_KEY_PERSONAL` が設定されている場合: **OpenAI API**
3. `GEMINI_API_KEY` が設定されている場合: **Gemini API**
いずれも設定されていない場合はAIによる処理は行われません。

## 使い方

### Google Apps Scriptのトリガー設定
スクリプトを自動実行するために、以下の2つのトリガーを設定します。

#### 1. デイリートリガー (RSS収集と見出し生成)
- **目的**: 毎日RSSフィードを収集し、AIによる見出し生成を行います。
- **実行する関数**: `mainAutomationFlow`
- **イベントのソース**: `時間主導型` > `日ベースのタイマー`
- **時刻**: 任意の時間帯（例: `午前 0 時～1 時`）

#### 2. ウィークリートリガー (週次ダイジェスト作成と通知)
- **目的**: 週に一度ダイジェストを作成し、HTMLメールで通知します。
- **実行する関数**: `weeklyDigestJob`
- **イベントのソース**: `時間主導型` > `週ベースのタイマー`
- **曜日**: 任意の曜日（例: `月曜日`）
- **時刻**: 任意の時間帯（例: `午前 9 時～10 時`）

## 注意事項
- **APIキーの管理**: APIキーは機密情報です。スクリプトプロパティに安全に保存し、コード内に直接書き込まないでください。
- **レート制限**: AIサービスや`UrlFetchApp`にはレート制限があります。APIの呼び出し頻度を適切に制御してください。
- **AIの出力**: AIが生成する見出しや要約は、常に完璧とは限りません。
- **スプレッドシートの保護**: `RSS` シートや `collect` シートのヘッダー行は、誤って変更されないように保護することを推奨します。

## 改善案
Daily処理の役割分担の最適化「Dailyはソース源からの記事集約と、必要に応じた要約に特化させる」という方針に賛成です。
これにより、日次の処理はシンプルで高速になり、特定のユーザーニーズ（キーワード分析、トレンド抽出など）への対応は、より高度な週次処理やオンデマンドのWebツールに集約できます。

処理フェーズ,現在の役割,提案する新しい役割,目的・効果
日次 (Daily),1. RSS収集・保存,1. RSS収集・保存 (A～C列),記事の網羅的な確保。
(mainAutomationFlow),2. LLMで見出し生成 (E列),"2. LLMで見出し/TL;DR生成 (E, H列)",記事内容の即時把握と、後の選定・分析のための生テキスト提供。
,3. LLMでAIスコア生成 (G列),3. AIスコアの生成を廃止/週次に移行,日次処理の高速化。スコアのプロンプト依存性解消。
週次 (Weekly),1. 記事フィルタリング,1. キーワードに基づく関連性フィルタリング,専門分野に合致する記事の絞り込み。
(weeklyDigestJob),2. ヒューリスティック/AI複合スコアで選定,2. 強化されたヒューリスティックスコアで選定,Keywordsシートを主軸とした選定の実現。
,3. LLMでレポート生成,3. LLMで「キーワード駆動型」トレンド分析レポート生成,ユーザー設定（キーワード）に合致した専門的な洞察の提供。
