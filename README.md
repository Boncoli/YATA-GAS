# RSS Collector & AI Digest

![Version](https://img.shields.io/badge/version-2.9.0-blue.svg)

## 概要

このGoogle Apps Script (GAS)プロジェクトは、指定したRSSフィードから自動で記事を収集し、Googleスプレッドシートに蓄積します。さらに、収集した記事に対してAI（Azure OpenAI, OpenAI, Gemini）を活用し、以下の4つの主要な機能を提供します。

1.  **AIによる見出し生成**: 収集した記事の要点を掴んだ、分かりやすい見出しを自動で生成します。
2.  **ダイジェストの自動生成・配信**: 「日刊ダイジェスト」と、先週からのトレンドの変化を分析する「週刊比較分析ダイジェスト」を自動で作成し、メールで配信します。
3.  **オンデマンドでの検索・分析**: Web UIからキーワードを入力し、関連する記事を即座に検索・分析します。

## 主な機能

- **RSSフィード自動収集**: 複数のRSSフィードから最新記事を巡回・収集し、重複を排除してスプレッドシートに保存します。
- **AI見出し生成**: 各記事のタイトルと本文の抜粋を基に、AIがキャッチーな1行見出しを生成します。
- **日刊ダイジェスト**: 過去24時間の**すべての記事**を対象に、AIが主要なトピックを抽出・要約して配信します。その日のニュース全体を俯瞰するのに役立ちます。
- **週刊比較分析ダイジェスト**:
  - **差分分析**: `Keywords`シートで指定したキーワードに基づき、AIが**先週の動向と比較**して「今週の新トピック」「深化した議論」などを分析します。
  - **要点サマリー**: 詳細な分析レポートの冒頭に、AIが生成した「今週のハイライト(tl;dr)」を付与。忙しい中でも重要な変化点を一目で把握できます。
- **Web UIによる検索・分析**:
  - スクリプトをWebアプリとしてデプロイすることで、専用のUIが利用可能になります。
  - キーワードを入力して、関連する記事の週刊ダイジェストをオンデマンドで生成できます。
  - AND検索（例: `AI 医療`）で記事を絞り込み、その結果をAIに即時分析させることも可能です。

## スプレッドシートの構成

このスクリプトを利用するには、6つのシートが必要です。

| シート名 | 目的 |
|---|---|
| `RSS` | 収集対象のRSSフィードURLを登録します。 |
| `collect` | 収集したすべての記事データが蓄積されます。 |
| `Keywords`| 週刊ダイジェストや記事のスコアリングに使うキーワードを管理します。 |
| `prompt` | 各機能でAIに指示を出すためのプロンプトテンプレートを管理します。 |
| `DigestHistory` | 週刊比較分析ダイジェストで生成された「要点サマリー」を記録・蓄積します。 |

<details>
<summary>各シートの詳細な列構成はこちら</summary>

### 1. `RSS` シート
| 列 | ヘッダー名 | 説明 |
|---|---|---|
| A | サイト名 | RSSフィードの提供元サイト名。 |
| B | RSS URL | RSSフィードのURL。 |

### 2. `collect` シート
| 列 | ヘッダー名 | 説明 |
|---|---|---|
| A | 日付 | 記事を収集した日時。 |
| B | 元タイトル | 記事の元のタイトル。 |
| C | URL | 記事のパーマリンク。 |
| D | 抜粋 | 記事の概要。HTMLタグは自動で除去されます。 |
| E | 見出し（AI生成）| AIが生成した1行見出し。 |
| F | ソース | 記事の提供元サイト名。 |

### 3. `Keywords` シート
| 列 | ヘッダー名 | 説明 |
|---|---|---|
| A | キーワード | 記事のフィルタリングに使うキーワード。`A and B`, `A or B` のような論理演算もサポート。 |
| B | 有効 | このキーワードを有効にする場合、任意の文字（例: `y`）を入力。空欄の場合は無効。 |
| C | 曜日 | トリガーによる自動実行時に、ここで指定した曜日にのみダイジェストを生成します。カンマ区切りで複数指定可（例: `月,水`）。`毎日`と指定するか空欄にすると、トリガー実行日に必ず評価されます。手動実行時はこの設定は無視されます。|

### 4. `prompt` シート
| 列 | ヘッダー名 | 説明 |
|---|---|---|
| A | キー | プロンプトの識別キー（例: `DAILY_DIGEST_SYSTEM`）。コード内で参照されます。 |
| B | 内容 | AIに渡すプロンプトの具体的な指示内容。 |

### 6. `DigestHistory` シート
| 列 | ヘッダー名 | 説明 |
|---|---|---|
| A | Date | 要点サマリーを生成した日時。 |
| B | Keyword | 対象となったキーワード。 |
| C | Summary | AIが生成したその週の「要点サマリー(tl;dr)」。 |

</details>

## セットアップ方法

### 1. スクリプトの配置
1.  Googleスプレッドシートを新規作成し、上記の6つのシートを準備します。
2.  「拡張機能」>「Apps Script」を選択してスクリプトエディタを開きます。
3.  `RSScollect.js`と`Index.html`の2つのファイルを作成し、それぞれの内容をエディタにコピー＆ペーストします。
4.  プロジェクトを保存します。

### 2. スクリプトプロパティの設定
スクリプトエディタの「プロジェクトの設定」（歯車アイコン）を開き、「スクリプト プロパティ」に以下の情報を設定します。

**最重要設定**
| プロパティ名 | 説明 |
|---|---|
| `EXECUTION_CONTEXT` | LLM呼び出しの優先順位を決めます。`COMPANY` (Azure優先) または `PERSONAL` (個人OpenAI優先) を設定します。**未設定の場合は`COMPANY`として動作します。** |

**APIキーとエンドポイント**
| プロパティ名 | 説明 |
|---|---|
| `OPENAI_API_KEY` | **Azure OpenAI の APIキー**を設定します。 |
| `OPENAI_API_KEY_PERSONAL` | **個人の OpenAI APIキー** (`sk-...`) を設定します。 |
| `GEMINI_API_KEY` | **Google AI Gemini の APIキー**を設定します。 |
| `AZURE_ENDPOINT_URL_NANO`| 軽量処理用のAzure OpenAIデプロイメントエンドポイントURL。 |
| `AZURE_ENDPOINT_URL_MINI` | 高次分析用のAzure OpenAIデプロイメントエンドポイントURL。 |

**ダイジェスト・メール設定**
| プロパティ名 | 説明 | デフォルト値 |
|---|---|---|
| `MAIL_TO` | ダイジェストメールの送信先アドレス。カンマ区切りで複数指定可。 | - |
| `MAIL_SENDER_NAME`| メールの送信者名。 | `RSS要約ボット` |
| `MAIL_SUBJECT_PREFIX` | メールの件名に付与される接頭辞。 | `【日刊/週間RSS】` |
| `DIGEST_SHEET_URL`| メールフッターに記載するスプレッドシートのURL。 | - |
| `DIGEST_DAYS` | 週刊ダイジェストの集計日数。 | `7` |
| `DIGEST_TOP_N` | 週刊ダイジェストでAI分析の対象とする上位記事数。| `20` |

**モデル・機能設定**
| プロパティ名 | 説明 | デフォルト値 |
|---|---|---|
| `OPENAI_MODEL_NANO` | 見出し生成などに使う軽量モデル名。 | `gpt-4.1-nano` |
| `OPENAI_MODEL_MINI` | ダイジェスト生成などに使う高次分析モデル名。| `gpt-4.1-mini` |

### 3. プロンプト設定例
`prompt`シートには、AIへの指示を柔軟に変更できるよう、プロンプトのテンプレートを登録します。
以下は、週刊比較分析ダイジェスト機能を利用するための推奨設定例です。

<details>
<summary>推奨プロンプト設定の詳細はこちら</summary>

#### 1. 比較分析ありの場合 (推奨・新規追加)
- **キー**: `TREND_USER_TEMPLATE_WITH_HISTORY`
- **内容**:
  ```
  # 指示
  あなたは指定されたキーワードの専門家として、最新の動向をまとめる週刊レポートを作成します。
  以下の「先週の要約」と「今週の記事リスト」を比較分析し、「今週のレポート」を生成してください。
  特に、先週からの変化点（新しい話題、深まった議論、話題性の低下など）を明確に記述してください。

  # 先週の要約
  {previous_summary}

  # 今週の記事リスト
  {article_list}

  # 出力形式
  【サマリー】
  ここに先週との比較を踏まえた今週の全体的な状況を2〜3文で記述。

  【新しいトピック】
  - 今週新たに登場した話題や、特に注目度が上がったテーマを箇条書きで記述。

  【継続・深化トピック】
  - 先週から引き続き議論されている重要なテーマについて、議論の進展や新しい視点を記述。

  【関連リンク】
  - 上記の分析の根拠となった、最も重要な記事のリンクを3つまでリストアップ。
  ```

#### 2. 比較分析なしの場合 (推奨・既存のものを更新)
- **キー**: `TREND_USER_TEMPLATE`
- **内容**:
  ```
  # 指示
  あなたは指定されたキーワードの専門家として、最新の動向をまとめる週刊レポートを作成します。
  以下の「今週の記事リスト」を分析し、「今週のレポート」を生成してください。

  # 今週の記事リスト
  {article_list}

  # 出力形式
  【サマリー】
  ここに今週の全体的な状況を2〜3文で記述。

  【注目トピック】
  - 今週特に注目すべき重要なテーマを箇条書きで記述。

  【関連リンク】
  - 上記の分析の根拠となった、最も重要な記事のリンクを3つまでリストアップ。
  ```
  > **Note:** 既存の`TREND_USER_TEMPLATE`がある場合、この例のように末尾に`{article_list}`プレースホルダーを追加すると、より安定的に動作します。

#### 3. 要点サマリー生成用 (推奨・新規追加)
- **キー**: `DIGEST_SUMMARY_SYSTEM`
- **内容**:
  ```
  以下のレポートテキスト全体を読み、最も重要なポイントを3つ抽出し、日本語の箇条書きで要約してください。各項目は50字以内で、具体的かつ簡潔に記述してください。前置きや後書きは一切不要です。

  出力形式:
  - 要点1...
  - 要点2...
  - 要点3...
  ```

</details>

### 4. トリガーの設定
スクリプトエディタの「トリガー」（目覚まし時計アイコン）から、以下の3つのトリガーを設定します。

| 実行する関数 | イベントのソース | タイマーの種類 | 推奨設定 | 目的 |
|---|---|---|---|---|
| `mainAutomationFlow` | 時間主導型 | 日ベース | 毎日 午前1時〜2時 | RSS収集、AI見出し生成 |
| `dailyDigestJob` | 時間主導型 | 日ベース | 毎日 午前8時〜9時 | 日刊ダイジェストの生成・配信 |
| `weeklyDigestJob`| 時間主導型 | 週ベース | 毎週月曜日 午前9時〜10時| 週刊ダイジェストの生成・配信 |

### 4. Web UIのデプロイ
1. スクリプトエディタ右上の「デプロイ」>「新しいデプロイ」を選択します。
2. 「種類の選択」で「ウェブアプリ」を選択します。
3. 「アクセスできるユーザー」を「自分のみ」または適切な範囲に設定し、「デプロイ」をクリックします。
4. 表示されたURLにアクセスすると、キーワード検索・分析UIが利用できます。

## アーキテクチャの概要
このスクリプトは、責務の分離と設定の一元化を重視して設計されています。
- **`AppConfig` モジュール**: スクリプトプロパティや固定設定をすべて集約し、初回実行時にキャッシュします。これにより、パフォーマンスを向上させつつ、設定へのアクセスを容易にしています。
- **`LlmService` モジュール**: Azure, OpenAI, Geminiといった複数のLLM呼び出しに関する複雑なロジック（フォールバック、プロンプト構築など）をすべてカプセル化しています。週刊ダイジェストでは**2段階のLLM呼び出し**（比較分析→要点サマリー生成）を行い、より質の高いインサイトを生成します。
- **機能ごとの関数群**: `SECTION`コメントにより、トリガー、ダイジェスト処理、トレンド検出、ユーティリティなどの機能ブロックごとにコードが整理されており、可読性と保守性が高められています。
