# RSScollect - Google Apps ScriptによるRSSフィード収集・AI見出し生成・週次ダイジェスト

![Version](https://img.shields.io/badge/version-2.6.0-blue.svg) ![Date](https://img.shields.io/badge/date-2025--11--10-lightgrey.svg)

## 概要
このGoogle Apps Script (GAS) プロジェクトは、指定されたRSSフィードから記事を自動的に収集し、Googleスプレッドシートに保存します。さらに、収集した記事の抜粋やタイトルを基に、AI (Gemini, Azure OpenAI, OpenAI) を利用して「ネットニュース風の見出し」を生成・追記します。

そして、週に一度、収集した記事の中から、`Keywords`シートで有効と設定されたキーワードに基づいて関連性の高いものをAIが自動で選別・要約し、見やすい**HTML形式のダイジェスト**としてメールで配信します。

## 機能
- **RSSフィード収集**: 登録されたRSSフィードから直近7日間の最新記事を自動取得し、重複を排除してスプレッドシートに追記します。
- **AI見出し生成**: 日次処理で、各記事のタイトルと抜粋を基に、AIが1行の見出しを生成し、E列に保存します。
  - 記事が短い、または抜粋がない場合、AI処理をスキップします。代わりに、記事タイトルが英語であれば`=GOOGLETRANSLATE`関数で日本語に翻訳し、日本語であればタイトルをそのまま見出しとして利用するフォールバック機能があります。
- **週次ダイジェスト (HTMLメール)**: `Keywords`シートで有効化されたキーワードに合致する記事を抽出し、ヒューリスティックなスコアに基づいて上位記事を選定。AIがそれらの記事を基にトレンドを分析・要約し、ダイジェストを作成します。
- **オンデマンド実行**: Webアプリとしてデプロイすることで、特定のキーワードを指定してダイジェストを即時生成することも可能です。
- **HTMLタグ除去**: 記事の抜粋から不要なHTMLタグを除去し、クリーンなテキストとして扱います。
- **トレンド検知**: 日次でキーワードの出現頻度を分析し、`Trends`シートに記録します。

## 週次ダイジェストの構成とデザイン
週次で配信されるHTMLメールは、以下のロジックとデザインで構成されています。

### ロジックのフロー
1. `weeklyDigestJob`が週次トリガーまたはWeb UI経由で実行されます。
2. 過去N日間の記事から、`Keywords`シートで有効化されたキーワードに合致する記事を抽出します。
3. 記事を総合的にランク付けします。スコアは「キーワードのマッチ数」「記事の鮮度」「抜粋の有無」を基に計算され、より関連性が高く新しい記事が優先されます。
4. LLM（AI）が、選定された上位記事群をキーワードごとに分類・分析し、**Markdown形式**でダイジェスト本文（主要トレンド）を生成します。
5. 生成されたMarkdownは、`markdownToHtml`関数によって見やすい**HTML形式**に変換されます。
6. 最終的に`GmailApp`を通じてHTMLメールとして配信されます。

### メールの構成
メールは以下のセクションで構成され、情報を直感的に把握できます。
1. **今週の注目キーワード**: ダイジェストで取り上げられたキーワードと、それぞれのヒット記事数を表示します。
2. **主要トレンド**: AIがキーワードごとにトレンドを分析し、それぞれの要点と代表記事リンクをまとめたセクション。
3. **フッター**: 全記事を確認できるスプレッドシートへのリンク。

### HTMLデザイン
- **フォント**: メイリオやヒラギノ角ゴなど、PCでもスマートフォンでも読みやすい標準的な日本語フォントを使用します。
- **レイアウト**: 白を基調とした背景に、ダークグレーのテキストを配置した、シンプルでクリーンなデザインです。
- **装飾**: `markdownToHtml`関数により、以下のMarkdown記法がHTMLに変換され、視覚的に分かりやすいスタイルが適用されます。
  - `### 見出し` → `<h3>`
  - `**太字**` → `<b>`
  - `*[リンク](URL)*` → `<a href="...">`
  - `---` → `<hr>` (水平線)
  - `- リスト` → `•` (箇条書き)

## スプレッドシートの構造

### 1. `RSS` シート (RSSフィードリスト)
| 列 | ヘッダー名 | 説明 |
|---|---|---|
| A | サイト名 | RSSフィードの提供元サイト名 |
| B | RSS URL | RSSフィードのURL |

### 2. `collect` シート (収集データ)
| 列 | ヘッダー名 | 説明 |
|---|---|---|
| A | 日付 | 記事の公開日時 (pubDate/updated) |
| B | 元タイトル | 記事の元のタイトル |
| C | URL | 記事のURL |
| D | 抜粋 | 記事の抜粋または概要 (HTMLタグ除去済み) |
| E | 見出し（AI生成）| AIが生成したネットニュース風の見出し |
| F | ソース | 記事のソースサイト名 (RSSシートのサイト名) |

### 3. `Keywords` シート (キーワードリスト)
| 列 | ヘッダー名 | 説明 |
|---|---|---|
| A | キーワード | 記事選定に利用するキーワード (`A and B`, `A or B` の形式も可) |
| B | 有効 | このキーワードを利用する場合、任意の文字を入力（例: `Y`）。空欄の場合は無効。 |

### 4. `prompt` シート (AIプロンプト設定)
| 列 | ヘッダー名 | 説明 |
|---|---|---|
| A | キー | プロンプトの識別キー (例: `DAILY_SYSTEM`, `TREND_USER_TEMPLATE`) |
| B | 内容 | AIに渡すプロンプトの本文 |

### 5. `Trends` シート (トレンドデータ)
| 列 | ヘッダー名 | 説明 |
|---|---|---|
| A | 日付 | トレンドを記録した日付 |
| B | キーワード（英語） | トレンドとして検出されたキーワード（英語） |
| C | キーワード（日本語） | AIが自動翻訳した日本語訳 |
| D | 出現回数 | そのキーワードの出現回数 |
| E | 変化率 | 前週比での変化率（`New`：新規、または`+100%`などの増減率） |
| F | 関連記事数 | そのキーワードに関連する記事の数 |
| G | 要約 | キーワードの簡潔な説明（今後拡張用） |

## 設定方法

### 1. Google Apps Script プロジェクトの作成
1. Googleスプレッドシートを開き、「拡張機能」>「Apps Script」を選択してスクリプトエディタを開きます。
2. `RSScollect.js` の内容をスクリプトエディタにコピー＆ペーストします。現在は単一のファイルで管理されています。
3. プロジェクト名を「RSScollect」など、任意の分かりやすい名前に変更します。

### 2. スクリプトプロパティの設定
スクリプトエディタの左側メニューにある「プロジェクトの設定」（歯車アイコン）をクリックし、「スクリプトプロパティ」セクションで以下のプロパティを設定します。

| プロパティ名 | 説明 | 設定例 | 必須/任意 |
|---|---|---|---|
| `GEMINI_API_KEY` | Gemini APIを利用する場合のAPIキー | `AIzaSy...` | AI利用時 |
| `AZURE_ENDPOINT_URL_NANO` | Azure OpenAI（軽量処理用）のエンドポイントURL | `https://your-resource.openai.azure.com/...` | AI利用時 |
| `AZURE_ENDPOINT_URL_MINI` | Azure OpenAI（高次分析用）のエンドポイントURL | `https://your-resource.openai.azure.com/...` | AI利用時 |
| `OPENAI_API_KEY` | Azure OpenAIを利用する場合のAPIキー | `your-azure-openai-api-key` | AI利用時 |
| `OPENAI_API_KEY_PERSONAL` | 個人的なOpenAI APIを利用する場合のAPIキー | `sk-...` | AI利用時 |
| `OPENAI_MODEL_NANO` | **軽量処理**で使うOpenAI/Azureモデル名（見出し生成、キーワード抽出等） | `gpt-4.1-nano` (既定値) | 任意 |
| `OPENAI_MODEL_MINI` | **高次分析処理**で使うOpenAI/Azureモデル名（日刊、週刊、検索分析等） | `gpt-4.1-mini` (既定値) | 任意 |
| `DIGEST_DAYS` | 週次ダイジェストの集計日数 | `7` (既定値) | 任意 |
| `DIGEST_TOP_N` | 週次ダイジェストでAI分析の対象とする上位記事数 | `30` (既定値) | 任意 |
| `NOTIFY_CHANNEL_WEEKLY` | 週次ダイジェストの通知先 (`email` または `none`) | `email` (既定値) | 任意 |
| `MAIL_TO` | メール通知の送信先メールアドレス (カンマ区切りで複数指定可) | `your-email@example.com` | `NOTIFY_CHANNEL_WEEKLY` が `email` の場合必須 |
| `MAIL_SUBJECT_PREFIX` | メール件名のプレフィックス | `【週間RSS】` (既定値) | 任意 |
| `MAIL_SENDER_NAME` | メール送信者名 | `RSS要約ボット` (既定値) | 任意 |
| `TREND_DETECTION_ENABLED` | トレンド検出機能の有効/無効 | `true` / `false` | 任意 |

**注意:** AIモデルの選択は以下の優先順位で行われます。
1. **Azure OpenAI**: `AZURE_ENDPOINT_URL_NANO` または `AZURE_ENDPOINT_URL_MINI` と `OPENAI_API_KEY` が設定されている場合。
2. **OpenAI (個人)**: `OPENAI_API_KEY_PERSONAL` が設定されている場合。
3. **Gemini API**: `GEMINI_API_KEY` が設定されている場合。

いずれも設定されていない場合はAIによる処理は行われません。

**モデル選択基準:**
- **NANO モデル** (`gpt-4.1-nano`): 記事見出し生成、キーワード抽出など、単純な処理用。
- **MINI モデル** (`gpt-4.1-mini`): 日刊・週刊ダイジェスト、記事分析など、高次な分析が必要な処理用。
- **Gemini API**: コード内で `gemini-2.5-flash-lite` に設定されています。

## 使い方

### Google Apps Scriptのトリガー設定
スクリプトを自動実行するために、以下の2つのトリガーを設定します。

#### 1. デイリートリガー (RSS収集と見出し生成)
- **目的**: 毎日RSSフィードを収集し、AIによる見出し生成を行います。
- **実行する関数**: `mainAutomationFlow`
- **イベントのソース**: `時間主導型` > `日ベースのタイマー`
- **時刻**: 任意の時間帯（例: `午前 0 時～1 時`）

#### 2. ウィークリートリガー (週次ダイジェスト作成と通知)
- **目的**: 週に一度ダイジェストを作成し、HTMLメールで通知します。
- **実行する関数**: `weeklyDigestJob`
- **イベントのソース**: `時間主導型` > `週ベースのタイマー`
- **曜日**: 任意の曜日（例: `月曜日`）
- **時刻**: 任意の時間帯（例: `午前 9 時～10 時`）

## アーキテクチャ
このスクリプトは、日次処理と週次処理の2つの主要なフローで動作します。コードは単一の`RSScollect.js`ファイル内に、責務ごとにコメントで論理的に分割されており、メンテナンス性が高められています。

- **`Config`オブジェクト**: シート名、API設定、各種定数などを一元管理する`Config`オブジェクトをコード冒頭で定義しており、設定の変更が容易です。

### 処理の役割分担
| 処理フェーズ | 実行関数 | 役割 | 目的・効果 |
|---|---|---|---|
| **日次 (Daily)** | `mainAutomationFlow` | 1. **RSS収集・保存**: 新しい記事を収集し、重複を除いてスプレッドシートに保存します。<br>2. **AI見出し生成**: 記事ごとにAIが見出しを生成し、E列に保存します。<br>3. **トレンド検知**: キーワードの出現頻度を記録します。 | 記事を網羅的に確保し、後の分析で利用できるよう事前に内容を要約しておくことで、週次処理の負荷を軽減し高速化します。 |
| **週次 (Weekly)** | `weeklyDigestJob` | 1. **キーワードフィルタリング**: `Keywords`シートに基づき、関連性の高い記事群を抽出します。<br>2. **スコアリングと選定**: ヒューリスティックスコアで記事をランク付けし、ダイジェストに載せる記事を選定します。<br>3. **ダイジェスト生成・配信**: 選定された記事を基に、AIがトレンドを分析し、HTML形式のメールを生成・配信します。 | ユーザーが設定したキーワードに合致する、専門的で価値の高い情報を抽出し、ダイジェストとして提供します。 |
