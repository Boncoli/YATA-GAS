# RSS Collector & AI Digest

![Version](https://img.shields.io/badge/version-2.8.0-blue.svg)

## 概要

このGoogle Apps Script (GAS)プロジェクトは、指定したRSSフィードから自動で記事を収集し、Googleスプレッドシートに蓄積します。さらに、収集した記事に対してAI（Azure OpenAI, OpenAI, Gemini）を活用し、以下の3つの主要な機能を提供します。

1.  **AIによる見出し生成**: 収集した記事の要点を掴んだ、分かりやすい見出しを自動で生成します。
2.  **ダイジェストの自動生成・配信**: 「日刊」「週刊」の2種類のダイジェストを自動で作成し、メールで配信します。
3.  **オンデマンドでの検索・分析**: Web UIからキーワードを入力し、関連する記事を即座に検索・分析します。

## 主な機能

- **RSSフィード自動収集**: 複数のRSSフィードから最新記事を巡回・収集し、重複を排除してスプレッドシートに保存します。
- **AI見出し生成**: 各記事のタイトルと本文の抜粋を基に、AIがキャッチーな1行見出しを生成します。
- **日刊ダイジェスト**: 過去24時間の**すべての記事**を対象に、AIが主要なトピックを抽出・要約して配信します。その日のニュース全体を俯瞰するのに役立ちます。
- **週刊ダイジェスト**: `Keywords`シートで指定したキーワードに基づき、関連性の高い記事をAIが深掘り分析。注目すべきトレンドをセクションごとにまとめて配信します。
- **トレンドキーワード検出**: 日々の記事から注目すべきキーワードを自動で抽出し、`Trends`シートに記録。新規出現（🆕）か既出（✓）かも判別します。
- **Web UIによる検索・分析**:
  - スクリプトをWebアプリとしてデプロイすることで、専用のUIが利用可能になります。
  - キーワードを入力して、関連する記事の週刊ダイジェストをオンデマンドで生成できます。
  - AND検索（例: `AI 医療`）で記事を絞り込み、その結果をAIに即時分析させることも可能です。

## スプレッドシートの構成

このスクリプトを利用するには、5つのシートが必要です。

| シート名 | 目的 |
|---|---|
| `RSS` | 収集対象のRSSフィードURLを登録します。 |
| `collect` | 収集したすべての記事データが蓄積されます。 |
| `Keywords`| 週刊ダイジェストや記事のスコアリングに使うキーワードを管理します。 |
| `prompt` | 各機能でAIに指示を出すためのプロンプトテンプレートを管理します。 |
| `Trends` | トレンドキーワードの出現頻度などが記録されます。 |

### 1. `RSS` シート

| A | サイト名 | RSSフィードの提供元サイト名。 |

### 2. `collect` シート
| 列 | ヘッダー名 | 説明 |
|---|---|---|
| A | 日付 | 記事を収集した日時。 |
| B | 元タイトル | 記事の元のタイトル。 |

| D | 抜粋 | 記事の概要。HTMLタグは自動で除去されます。 |
| E | 見出し（AI生成）| AIが生成した1行見出し。 |
| F | ソース | 記事の提供元サイト名。 |

### 3. `Keywords` シート
| 列 | ヘッダー名 | 説明 |
|---|---|---|
| A | キーワード | 記事のフィルタリングに使うキーワード。`A and B`, `A or B` のような論理演算もサポート。 |
| B | 有効 | このキーワードを有効にする場合、任意の文字（例: `y`）を入力。空欄の場合は無効。 |

### 4. `prompt` シート
| 列 | ヘッダー名 | 説明 |
|---|---|---|
| A | キー | プロンプトの識別キー（例: `DAILY_DIGEST_SYSTEM`）。コード内で参照されます。 |

### 5. `Trends` シート
| 列 | ヘッダー名 | 説明 |
|---|---|---|
| A | 日付 | トレンドを記録した日付。 |
| B | キーワード | トレンドとして検出されたキーワード。 |
| C | キーワード（日本語）| `=GOOGLETRANSLATE`による自動翻訳。 |
| D | 出現回数 | その日の記事内での出現回数。 |
| E | 変化率 | **`🆕 New`** (過去7日以内に出現なし) または **`✓ 既出`** を表示。 |
| F | 関連記事数 | （出現回数と同じ値が入ります） |
| G | 要約 | （将来的な拡張用のための空欄） |

## セットアップ方法

### 1. スクリプトの配置
1.  Googleスプレッドシートを新規作成し、上記の5つのシートを準備します。
2.  「拡張機能」>「Apps Script」を選択してスクリプトエディタを開きます。
3.  `RSScollect.js`と`Index.html`の2つのファイルを作成し、それぞれの内容をエディタにコピー＆ペーストします。
4.  プロジェクトを保存します。

### 2. スクリプトプロパティの設定
スクリプトエディタの「プロジェクトの設定」（歯車アイコン）を開き、「スクリプト プロパティ」に以下の情報を設定します。

**最重要設定**
| プロパティ名 | 説明 |
|---|---|
| `EXECUTION_CONTEXT` | LLM呼び出しの優先順位を決めます。`COMPANY` (Azure優先) または `PERSONAL` (個人OpenAI優先) を設定します。**未設定の場合は`COMPANY`として動作します。** |

**APIキーとエンドポイント**
| プロパティ名 | 説明 |
|---|---|
| `OPENAI_API_KEY` | **Azure OpenAI の APIキー**を設定します。 |
| `OPENAI_API_KEY_PERSONAL` | **個人の OpenAI APIキー** (`sk-...`) を設定します。 |
| `GEMINI_API_KEY` | **Google AI Gemini の APIキー**を設定します。 |
| `AZURE_ENDPOINT_URL_NANO`| 軽量処理用のAzure OpenAIデプロイメントエンドポイントURL。 |
| `AZURE_ENDPOINT_URL_MINI` | 高次分析用のAzure OpenAIデプロイメントエンドポイントURL。 |

**ダイジェスト・メール設定**
| プロパティ名 | 説明 | デフォルト値 |
|---|---|---|
| `MAIL_TO` | ダイジェストメールの送信先アドレス。カンマ区切りで複数指定可。 | - |
| `MAIL_SENDER_NAME`| メールの送信者名。 | `RSS要約ボット` |
| `MAIL_SUBJECT_PREFIX` | メールの件名に付与される接頭辞。 | `【日刊/週間RSS】` |
| `DIGEST_SHEET_URL`| メールフッターに記載するスプレッドシートのURL。 | - |
| `DIGEST_DAYS` | 週刊ダイジェストの集計日数。 | `7` |
| `DIGEST_TOP_N` | 週刊ダイジェストでAI分析の対象とする上位記事数。| `20` |

**モデル・機能設定**
| プロパティ名 | 説明 | デフォルト値 |
|---|---|---|
| `OPENAI_MODEL_NANO` | 見出し生成などに使う軽量モデル名。 | `gpt-4.1-nano` |
| `OPENAI_MODEL_MINI` | ダイジェスト生成などに使う高次分析モデル名。| `gpt-4.1-mini` |
| `TREND_DETECTION_ENABLED`| トレンド検出機能を有効にする場合は `true` を設定。| `false` |

### 3. トリガーの設定
スクリプトエディタの「トリガー」（目覚まし時計アイコン）から、以下の3つのトリガーを設定します。

| 実行する関数 | イベントのソース | タイマーの種類 | 推奨設定 | 目的 |
|---|---|---|---|---|
| `mainAutomationFlow` | 時間主導型 | 日ベース | 毎日 午前1時〜2時 | RSS収集、AI見出し生成、トレンド検出 |
| `dailyDigestJob` | 時間主導型 | 日ベース | 毎日 午前8時〜9時 | 日刊ダイジェストの生成・配信 |
| `weeklyDigestJob`| 時間主導型 | 週ベース | 毎週月曜日 午前9時〜10時| 週刊ダイジェストの生成・配信 |

### 4. Web UIのデプロイ
1. スクリプトエディタ右上の「デプロイ」>「新しいデプロイ」を選択します。
2. 「種類の選択」で「ウェブアプリ」を選択します。
3. 「アクセスできるユーザー」を「自分のみ」または適切な範囲に設定し、「デプロイ」をクリックします。
4. 表示されたURLにアクセスすると、キーワード検索・分析UIが利用できます。

## アーキテクチャの概要
このスクリプトは、責務の分離と設定の一元化を重視して設計されています。
- **`AppConfig` モジュール**: スクリプトプロパティや固定設定をすべて集約し、初回実行時にキャッシュします。これにより、パフォーマンスを向上させつつ、設定へのアクセスを容易にしています。
- **`LlmService` モジュール**: Azure, OpenAI, Geminiといった複数のLLM呼び出しに関する複雑なロジック（フォールバック、プロンプト構築など）をすべてカプセル化しています。他の関数は `LlmService.summarize()` のように、目的を伝えるだけでAIの機能を利用できます。
- **機能ごとの関数群**: `SECTION`コメントにより、トリガー、ダイジェスト処理、トレンド検出、ユーティリティなどの機能ブロックごとにコードが整理されており、可読性と保守性が高められています。
