# RSS Collector & AI Intelligence Tool

![Version](https://img.shields.io/badge/version-3.0.0-blue.svg)

## 概要

このプロジェクトは、単なる「RSSリーダー」や「要約ツール」ではありません。
臨床検査・バイオ技術などの専門領域において、**「Webでの能動的な深掘り（Deep Dive）」**と**「週次の定点観測（Trend Tracking）」**を両立させる、**AI駆動型のインテリジェンス・ツール**です。

膨大なニュース記事を自動収集・蓄積し、LLM（GPT-4o等）が「新規技術の発見」や「トレンドの進捗（加速・停滞）」を分析して可視化します。

## コンセプトと提供価値

1.  **「点」ではなく「線」を見る（週刊トレンド分析）**
    * 過去の分析結果をシステムが記憶。先週の状況と今週のニュースをAIが突合し、「あの治験が進んだ」「この話題は沈静化した」といった**時間軸での変化**をレポートします。
2.  **「今」の「新規性」を掘り起こす（Webスポット検索）**
    * 会議前やアイデア出しの瞬間に、直近（約1ヶ月分）のデータから**「最新の技術的成果」や「重要な知見」**だけを抽出。リッチなHTMLレポートで即座に状況を把握できます。
3.  **持続可能な運用（メンテナンスフリー）**
    * 日々のノイズ（日刊メール）を遮断し、古いデータは自動で削除。情報の洪水に溺れず、本当に重要なインサイトだけを受け取る設計です。

## 主な機能

### 1. Web UIによるスポット分析（Deep Dive）
* **目的**: 能動的なリサーチ。直近の「新規技術」や「ブレイクスルー」の発掘。
* **特徴**:
    * キーワードに関連する**直近60件**（約1ヶ月分相当）の記事を総ざらい分析。
    * **「新規性（Novelty）」**にフォーカスし、既存技術との違いやインパクトを抽出。
    * CSS分離型の軽量HTMLによる、視認性の高いカード形式レポート。

### 2. 週刊トレンドレポート（Trend Tracking）
* **目的**: 受動的な定点観測。登録キーワードの「変化」の把握。
* **特徴**:
    * **毎週月曜日**に、キーワードごとの動向レポートをメール配信。
    * **記事数厳選（Top 30）**: メールで読み切れる量に情報を凝縮。
    * **進捗タグ**: `[⚡ 新規]` `[🚀 進展]` `[➡️ 継続]` などのタグで、ステータス変化を一目で理解可能。

### 3. 自動メンテナンス
* **目的**: スプレッドシートのパフォーマンス維持。
* **特徴**:
    * 設定期間（デフォルト12ヶ月）を過ぎた古い記事を自動的に削除。

## システム構成

必要なスプレッドシート（6シート）:

| シート名 | 目的 |
|---|---|
| `RSS` | 収集対象のRSSフィードURL一覧。 |
| `collect` | 収集した全記事データ（データベース）。 |
| `Keywords`| 週刊レポートの観測対象キーワード。 |
| `prompt` | AIへの指示（プロンプト）テンプレート。 |
| `DigestHistory` | 週ごとの分析結果を蓄積し、次週の比較に使用。 |
| `Users` / `Trends` | (旧仕様互換用 / 一時データ用) |

## セットアップ手順

### 1. スクリプトプロパティの設定
「プロジェクトの設定」>「スクリプト プロパティ」に以下を設定します。

| プロパティ名 | 推奨設定値 | 説明 |
|---|---|---|
| `DIGEST_TOP_N` | **30** | 週刊メールの1キーワードあたりの分析対象数。メールの可読性を守るための黄金比。 |
| `DIGEST_DAYS` | `7` | 週刊レポートの集計期間。 |
| `OPENAI_MODEL_MINI` | `gpt-4o-mini` | 高速・安価・高性能な分析用モデル。 |
| `EXECUTION_CONTEXT` | `COMPANY` | (任意) Azure OpenAI優先なら`COMPANY`。 |

※ APIキー類（`OPENAI_API_KEY`, `AZURE_ENDPOINT...` 等）も適切に設定してください。

### 2. コードとファイルの修正（v3.0.0対応）
本機能を利用するには、`RSScollect.js` および `Index.html` に以下の修正が必要です。

* **RSScollect.js**:
    * Web検索の上限設定: `const WEB_ANALYSIS_LIMIT = 60;`
    * トークン上限緩和: `LlmService` 呼び出し時に `maxTokens: 4096` を指定。
    * メンテナンス関数: `maintenanceDeleteOldArticles` の追加。
* **Index.html**:
    * `<style>`タグ内に、レポート表示用のCSSクラスを追加（CSS分離対応）。

### 3. プロンプトの登録（`prompt` シート）
「インテリジェンス・ツール」として機能させるため、以下のプロンプトセットを登録してください。

* `WEB_ANALYSIS_SYSTEM`: 新規技術発掘・HTML出力用。
* `TREND_SYSTEM`: 変化のベクトル（速度・影響）分析用。
* `TREND_USER_TEMPLATE_WITH_HISTORY`: 履歴比較・進捗タグ付け用。
* `DIGEST_SUMMARY_SYSTEM`: 次週用データ保存（数値・固有名詞維持）用。
* `BATCH_SYSTEM` / `USER`: 基礎データ構造化用。

### 4. トリガーの設定（推奨運用フロー）
「日刊」を停止し、「週刊」と「Web検索」にリソースを集中させます。

| 関数名 | イベント | タイマー設定 | 目的 |
|---|---|---|---|
| `mainAutomationFlow` | 時間主導 | 毎日 1:00〜2:00 | RSS収集・AI基礎解析（必須） |
| `weeklyDigestJob` | 時間主導 | 毎週 月曜 9:00〜10:00 | 週刊トレンドレポート配信 |
| `maintenanceDeleteOldArticles` | 時間主導 | 毎週 日曜 3:00〜4:00 | 古いデータの自動削除 |
| **(削除)** `dailyDigestJob` | - | - | **設定しない（ノイズ削減のため停止）** |

## 運用ベストプラクティス

1.  **キーワードの選定**:
    * いきなり `Keywords` シートに登録せず、まず **Web UI** で検索を試す。
    * ヒット数が **50〜80件前後** になるよう `AND` 検索などで絞り込む（例: `AI` → `AI and 創薬`）。
    * 「いい感じ」の結果が出たらシートに登録する。
2.  **使い分け**:
    * **Web UI**: 「最近どう？」と思ったら使う（能動的）。過去1ヶ月の**新規技術**が見つかる。
    * **メール**: 月曜朝にサッと読む（受動的）。先週からの**トレンド変化**が分かる。
3.  **データ保持**:
    * 初期設定では **12ヶ月** で自動削除されます。動作が重い場合はコード内の `KEEP_MONTHS` を `6` などに短縮してください。
